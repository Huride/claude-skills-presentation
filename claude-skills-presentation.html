  • 매번 다른 품질의 결과물

😓 실제 상황 예시

매번 반복되는 비효율적인 프로세스

사용자: "Jira에서 이번 스프린트 데이터 가져와서 회고 리포트 만들어줘" Claude: "회고 리포트를 만들어드리겠습니다. 어떤 형식을 원하시나요?" 사용자: "스프린트 목표 달성도, 번다운 차트, 속도 계산, 블로커 분석... 그리고 우리 팀은 보수적인 톤을 선호하고..." → 매번 긴 설명이 필요 😵 Claude: "이해했습니다. 범용적인 회고 리포트를 생성합니다..." 결과: 🤷‍♂️ 그저 그런 일반적인 리포트

📉 기존 방식의 한계

매 요청마다: "Jira 스프린트 데이터를 가져와서 다음 구조로 회고 리포트를 만들어줘: 1. 스프린트 목표 달성도 2. 번다운 차트 분석 3. 완료/미완료 티켓 분석 4. 버그 발생 현황 5. 팀 속도(Velocity) 계산 6. 블로커 분석 7. 다음 스프린트 개선점 그리고 우리 팀은 보수적인 톤을 선호하고..." → 매번 긴 컨텍스트 설명 필요 😵

--- name: "스프린트 회고 자동화" description: "Jira 데이터로 팀 회고 리포트 생성" --- # 스프린트 회고 Skills ## 목적 - 스프린트 완료 후 자동 회고 리포트 생성 - 팀 성과 분석 및 개선점 도출 - 경영진/개발팀별 맞춤 리포트 ## 사용법 사용자가 "스프린트 회고 만들어줘"라고 하면: 1. Jira에서 스프린트 데이터 수집 2. template.md 기반 리포트 생성 3. metrics.py로 성과 지표 계산 ## 출력 형식 - 목표 달성도: X% - 완료 스토리 포인트: X/Y - 주요 성과 및 블로커 분석
